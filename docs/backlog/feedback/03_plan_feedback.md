# Plan Feedback

## 1. 日期输入不友好

- 问题：编辑目标时，开始日期和目标日期必须手动填写固定 `YYYY-MM-DD` 格式，使用不友好。
- 方案：改为年/月/日三个独立组件，支持双击进入手动填写，以及下拉选择年、月、日。
- 状态：Sprint 4C 实现。

## 2. 目标层级缺少“自定义”

- 问题：当前 `goalLevel` 只有 `life/year/quarter/month/week/day`，无法表达自定义周期目标。
- 方案：增加 `custom / 自定义`。只有 custom 层级下，目标日期可手动编辑；非 custom 层级下，目标日期根据开始日期和层级自动计算。
- 状态：Sprint 4C 实现。

## 3. 目标日期应根据目标层级自动计算

- 问题：新建目标时，用户希望选择目标层级后，开始日期默认今天，目标日期自动根据层级计算。
- 方案：新建目标时 `startDate` 默认今天；非 custom 层级按 `startDate + goalLevel` 自动计算；修改开始日期或层级时自动重算；custom 层级保留用户手动输入。
- 状态：Sprint 4C 实现。

## 4. “排序”字段语义工程化

- 问题：UI 中直接暴露“排序”不自然。
- 方案：UI 改名为“优先级”，并说明“数值越小越靠前”。数据库字段继续使用 `sortOrder`。
- 状态：Sprint 4C 实现。

## 5. 缺少删除计划功能

- 问题：当前 Plan 只能将目标状态设为“已取消”，但没有真正的删除入口。
- 影响：用户无法清理误创建、废弃或测试目标。
- 短期方案：增加软删除功能，UI 中提供删除入口，删除前二次确认。
- 长期方案：后续可增加回收站或恢复删除。
- 状态：Sprint 4D 实现。

## 6. 手动状态设置不符合使用直觉

- 问题：当前“未开始 / 进行中 / 已完成 / 暂停 / 已取消”都依赖手动选择，操作鸡肋且不符合目标管理直觉。
- 影响：用户需要额外维护状态，增加输入负担。
- 短期方案：不再把 status 作为主要 UI 操作；根据开始日期、目标日期和今天日期自动显示未开始、进行中、已过期；完成与否改为 checkbox。
- 长期方案：后续可在需要时增加暂停、阻塞等高级状态。
- 状态：Sprint 4D 实现。

## 7. 日期选择组件响应延迟明显

- 问题：当前年/月/日组件响应有明显延迟。
- 原因推测：`DropdownButtonFormField` 在 build 时生成较多年份选项，组件树较重。
- 短期方案：替换为更轻量的日期部件；默认显示年/月/日三个按钮或紧凑字段；点击后使用底部弹层或轻量滚轮选择；双击仍支持手动输入。
- 状态：Sprint 4D 实现。

## 8. 缺少归档功能

- 问题：目标完成或过期后会长期堆积在列表中。
- 影响：Plan 会越来越乱，降低长期使用价值。
- 短期方案：增加归档功能；默认列表隐藏已归档目标；支持查看归档目标和从归档中恢复。
- 长期方案：后续可增加自动归档规则。
- 状态：Sprint 4D 实现。

## 9. 缺少分类、筛选、排序能力

- 问题：目标增多后，无法按层级、生命周期、优先级等方式查看。
- 影响：目标列表可读性下降。
- 短期方案：增加筛选条，支持按目标层级、生命周期筛选，显示或隐藏归档，并按优先级、目标日期、创建时间、层级排序。
- 长期方案：后续可增加搜索、标签、看板视图。
- 状态：Sprint 4D 实现。
