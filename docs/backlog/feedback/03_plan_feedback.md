# Plan Feedback

## 1. 日期输入不友好

- 问题：编辑目标时，开始日期和目标日期必须手动填写固定 `YYYY-MM-DD` 格式，使用不友好。
- 方案：改为年/月/日三个独立组件，支持双击进入手动填写，以及下拉选择年、月、日。
- 状态：Sprint 4C 实现。

## 2. 目标层级缺少“自定义”

- 问题：当前 `goalLevel` 只有 `life/year/quarter/month/week/day`，无法表达自定义周期目标。
- 方案：增加 `custom / 自定义`。只有 custom 层级下，目标日期可手动编辑；非 custom 层级下，目标日期根据开始日期和层级自动计算。
- 状态：Sprint 4C 实现。

## 3. 目标日期应根据目标层级自动计算

- 问题：新建目标时，用户希望选择目标层级后，开始日期默认今天，目标日期自动根据层级计算。
- 方案：新建目标时 `startDate` 默认今天；非 custom 层级按 `startDate + goalLevel` 自动计算；修改开始日期或层级时自动重算；custom 层级保留用户手动输入。
- 状态：Sprint 4C 实现。

## 4. “排序”字段语义工程化

- 问题：UI 中直接暴露“排序”不自然。
- 方案：UI 改名为“优先级”，并说明“数值越小越靠前”。数据库字段继续使用 `sortOrder`。
- 状态：Sprint 4C 实现。
