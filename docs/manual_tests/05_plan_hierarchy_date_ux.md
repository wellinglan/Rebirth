# Plan Hierarchy & Date UX Manual Test

1. 启动 App。
2. 进入 Plan。
3. 点击“新建目标”。
4. 确认开始日期默认为今天。
5. 确认默认目标层级为“月度”。
6. 确认目标日期自动为开始日期加一个自然月。
7. 将开始日期改为 `2026-03-31`，确认月度目标日期为 `2026-04-30`。
8. 切换目标层级为“季度”，确认目标日期自动调整为开始日期加三个自然月。
9. 切换目标层级为“自定义”，确认目标日期变为可编辑。
10. 使用年、月、日下拉组件选择日期。
11. 分别双击年、月、日并手动填写数值。
12. 将“优先级”填为 `-1`，确认显示校验错误且不能保存。
13. 创建一个根目标。
14. 点击根目标的“添加子目标”。
15. 填写并创建子目标。
16. 点击根目标的“子目标”进入 child view。
17. 确认刚创建的子目标出现在列表中。
18. 点击返回上一级，确认回到根目标列表。
19. 关闭并重新启动 App。
20. 确认根目标和子目标仍然存在。
21. 确认 Today 和 Journal 页面仍可正常打开。

## Android 竖屏布局检查

1. 在 Android 真机竖屏下打开“新建目标”或“编辑目标”。
2. 分别检查“开始日期”和“目标日期”的年、月、日输入控件。
3. 确认四位年份保持单行显示，月份和日期不会被压缩、截断或逐字换行。
4. 确认日期清除按钮不会挤压年、月、日输入控件。
5. 确认两个日期区块之间留有清晰间距，表单可正常滚动且没有横向溢出。

### 缺陷记录

- 缺陷 ID：`PLAN-ANDROID-DATE-LAYOUT-001`
- 严重程度：中等。功能仍可操作，但日期可读性和输入体验明显受损。
- 复现条件：Android 真机竖屏，打开 Plan 的目标新建或编辑 Dialog。
- 实际结果：年、月、日与清除按钮被固定在同一横向区域；年份输入框过窄，`2026` 显示为纵向字符。
- 期望结果：窄屏下日期控件应改为可读的响应式布局，四位年份保持单行，所有输入和操作按钮均不重叠、不截断。
- 当前结论：Plan Android 日期 UX 不通过；修复并完成 Android 真机回归前，不应将该检查标记为 PASS。
